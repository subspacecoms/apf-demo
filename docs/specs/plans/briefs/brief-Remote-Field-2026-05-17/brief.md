# Product Brief: Rig Harness

## Executive Summary

Rig Harness is an autonomous orchestration system designed to close the loop on agentic software development. By integrating a hierarchical task queue (GitHub Projects) with a GCP-native execution runner, the harness enables AI developer agents to pick tasks, implement code in isolated environments, run quality gates (tests and design reviews), and submit PRs that are automatically reviewed and merged. The system runs continuously until the software meets the "ready" criteria, enabling hands-free feature implementation.

## The Problem

Current AI developer agents operate as one-off assistants requiring human intervention to:
- Identify and assign the next task.
- Set up isolated development environments to avoid workspace pollution.
- Run tests and enforce code quality standards (e.g., SOLID principles).
- Manage the VCS workflow (creating branches, submitting PRs, and merging).

Without an automated harness to connect these steps, agentic development remains fragmented, slow, and dependent on constant human oversight.

## The Solution

Rig Harness solves this by providing a closed-loop execution environment:
- **Hierarchical Task Queue**: Backed by GitHub Issues/Projects, allowing users to define Epics, Stories, and Tasks with a simple UI.
- **GCP-Native Execution**: An ephemeral runner (Cloud Run Job) that claims tasks, spins up isolated containers, and executes the developer agent loop.
- **Hybrid Quality Gates**: Code quality is verified using a hybrid approach: automated test suites, static analysis tools, and LLM reviews guided by a structured SOLID checklist.
- **Serialized Merging**: A dedicated **Coordination Agent** manages a serial queue of PRs, resolving merge conflicts and verifying integration before merging them into the main branch.
- **Safety Valves**: Configurable iteration limits (default: 100 iterations) that pause the loop and request human review to prevent infinite agent run-time loops.

## What Makes This Different

- **Pure Ephemerality**: Most agentic loops run on persistent VMs. Rig Harness uses Cloud Run Jobs to spin up agents on-demand. This guarantees a clean slate (no state contamination) and scales costs to zero when no tasks are active.
- **VCS-PM Coherence**: By using GitHub Projects as the source of truth, the boundary between project management and development is seamless. Moving a card *is* the execution trigger.
- **Decoupled Verification**: Agent reasoning (Cloud Run) is decoupled from code verification (Cloud Build). Cloud Build runs standard CI/CD pipelines, ensuring agents are subjected to the same quality gates as human developers.
- **Agent Agnostic (Decoupled Orchestration)**: The harness executes agents via generic container interfaces. Developers can swap LLM models, prompts, or entire agent frameworks simply by updating the `rig.yaml` configuration.

## Who This Serves

- **Software Engineering Teams**: Looking to automate routine feature implementation, bug fixes, or dependency updates.
- **Product Managers**: Who want to see tasks implemented and merged simply by managing the Kanban board.
- **Solo Developers & Startups**: Looking to leverage a team of agents to accelerate development without managing infrastructure.

## Success Criteria

- **Zero-Touch Merges**: Successful implementation and merging of a task from "To Do" to `main` without human intervention.
- **High Build Success Rate**: PRs generated by agents pass Cloud Build sanity checks at a high rate (>80%).
- **Reduced Cycle Time**: Decrease in time from task assignment to PR merge for routine tasks.
- **Cost Efficiency**: Scale-to-zero compute costs when the task queue is empty.

## Scope

### In-Scope (V1)
- GitHub Webhook receiver (Cloud Run Service) to detect Project board movements.
- Agent Runner (Cloud Run Job) for the `Implementer` agent.
- Integration with Cloud Build for running tests and sanity checks.
- Automated PR creation.
- **Coordination Agent** (Cloud Run Job/Service) to serialize PR reviews, resolve conflicts, and merge them.
- **Merge Verification Agent** (Cloud Run Job) spawned by the Coordinator to run fast, local verification checks on rebased branches before merging.
- **Loop Control**: Configurable iteration counter that pauses agent execution and requests human review after X iterations.
- **Dynamic Environment Builder**: Pipeline to automatically rebuild the runner Docker image when package dependencies change.
- **Ambiguity Handling**: Two-tiered resolution flow for unclear task requirements (non-blocker best guesses vs. blocker halts).
- Support for hierarchical tasks (Epics -> Stories -> Tasks) in GitHub Projects.

### Out-of-Scope (V1)
- Self-healing infrastructure (agents modifying GCP configs).
- Support for non-GitHub VCS (GitLab, Bitbucket).
- Multi-repository orchestration (V1 is strictly mapped 1-to-1 with a single repository).
- Complex multi-agent negotiations (kept to linear handoffs).

## Vision

In 2-3 years, Rig Harness becomes the standard operating system for AI software development teams, allowing organizations to scale their development capacity by deploying autonomous agent squads that collaborate, implement, test, and deploy features with minimal human steering.
