# Product Brief: TaskFlow

## Executive Summary
TaskFlow is a minimalist, keyboard-first task manager designed for high-output professionals who find existing tools too bloated or too simple. It bridges the gap between a plain text file and a complex project management suite, offering lightning-fast entry and intelligent prioritization without the overhead of heavy configuration.

In a world of "notification fatigue," TaskFlow matters because it provides a "zen" environment for focus. It doesn't just store tasks; it helps users decide what to work on *next*, reducing the cognitive load of decision-making.

## The Problem
Most task managers suffer from "The Fiddling Trap"—users spend more time organizing, tagging, and color-coding tasks than actually doing them. 
- **Knowledge workers** feel overwhelmed by infinite backlogs.
- **Current tools** are either too "click-heavy" (SaaS apps) or lack the structure needed for team collaboration (plain text).
- **The Result:** Important tasks get buried, and the tool becomes a source of anxiety rather than a source of clarity.

## The Solution
TaskFlow provides a command-bar driven interface that allows for "thought-speed" task entry. 
- **Natural Language Input:** "Meeting with Sarah tomorrow at 2pm #project-alpha" automatically parses dates and projects.
- **The "Focus Flow" View:** A single-task interface that hides everything except the current priority.
- **Zero-Config Sync:** Instant persistence across devices with an offline-first architecture.

## What Makes This Different
- **Keyboard-First:** Every action can be performed without a mouse.
- **Opinionated Prioritization:** Instead of manual sorting, TaskFlow uses a "Urgency vs. Energy" matrix to suggest the best task for the user's current context.
- **No Moat (Honesty):** Our moat is execution speed and the "feel" of the UI. We aren't building a proprietary data silo; we are building the fastest way to get a thought into a system and out of your head.

## Who This Serves
- **The "Flow-State" Developer:** Needs to dump tasks quickly without leaving the terminal or IDE mindset.
- **The Overloaded Manager:** Needs to track dozens of small threads without losing the big picture.
- **Success Look Like:** The user closes TaskFlow feeling *less* stressed than when they opened it.

## Success Criteria
- **Input Velocity:** Average time from "intent to capture" to "task saved" < 3 seconds.
- **Retention:** 40% of users returning daily to clear their "Focus Flow."
- **NPS:** High "Love" score specifically for the UI/UX responsiveness.

## Scope
**In-Scope (V1):**
- Command-line style input bar.
- Basic project tagging.
- Focus mode (one task at a time).
- Local-first storage with cloud sync.

**Explicitly Out:**
- Complex Gantt charts or dependencies.
- Native mobile app (V1 will be a PWA).
- Integrated email/calendar (V1 focuses on task capture only).

## Vision
In 3 years, TaskFlow becomes the "operating system for intent." It won't just track what you *should* do, but will automate the "busy work" of setting up meetings and filing tickets, leaving the user with nothing but the creative work they love.
